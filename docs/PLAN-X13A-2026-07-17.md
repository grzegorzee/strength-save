**Goal:** Każdy aktywny dzień usera zostawia jeden dokument dzienny z licznikami: sesja aktywna, wyświetlenia ekranów, kluczowe akcje. Scheduled function zwija to nocą do `users/{uid}.activitySummary` (ostatnia aktywność, dni aktywne 7/30, treningi 7/30, top ekrany), żeby lista userów w panelu admina nie czytała setek dokumentów. Release train na końcu.

**Architektura podejścia:** Zero nowych kolekcji poza tym, co jest. Klient dopisuje NOWE nazwy liczników do istniejącego mechanizmu (koszt bez zmian: nadal 1 zapis per flush). Agregacja: scheduled function O(aktywnych wczoraj), zgodnie z zasadą architektoniczną z R2 (Functions tylko dla admin SDK, scheduled O(aktywnych)). Prywatność: liczniki zdarzeń, ZERO treści (nazw ćwiczeń, wag, notatek) i zero surowego clickstreamu.

**Tech stack:** React + TS, Firestore (rules z zamkniętymi schematami + testy `scripts/test-firestore-rules.mjs`, JDK21), Cloud Functions (wzorce: `functions/src/daily-reminder.ts`, testy vitest w functions/), vitest + Playwright mock.
