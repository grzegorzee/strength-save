5. `handleStartWorkout` (`WorkoutDay.tsx:1223-1392`):
   - sesja provisional/offline → `result.existing === false` → gałąź `else` (linia 1328): bezwarunkowy `setExerciseSets(prefilled)` + `persistDraftSnapshot(initialDraft)` z `version: 1` → **deterministyczne wymazanie draftu**;

Ścieżka "Kontynuuj trening" z Dashboardu jest bezpieczna, bo nawiguje BEZ `autostart` (`Dashboard.tsx:613-615`, `workout-resume.ts:24-25`). Dlatego bug objawia się tylko przy powrocie przez historię.

Pokrycie testami: ZERO. Żaden test (unit/e2e) nie robi sekwencji start → serie → /plan/edit → powrót.
