# PLAN X14C: import historii treningów ze Strong i Hevy (CSV) (2026-07-19)

> **Dla agenta wykonawcy:** WYMAGANY SUB-SKILL: `superpowers:executing-plans` (albo `superpowers:subagent-driven-development`). Kroki mają checkboxy `- [ ]`. TRYB 100% AUTONOMICZNY.
> Kontekst: wymaga WDROŻONYCH X14A i X14B (import korzysta z typów serii z X14B: duration/distance ze Strong CSV). Synteza researchy: import = warunek przejęcia usera z wieloletnią historią ("nie zacznę od zera"), funkcja akwizycyjna, DARMOWA.
> UWAGA NAJWYŻSZEJ WAGI: to plan piszący do danych usera. Dane usera są ŚWIĘTE. Każdy zapis przez dry-run + tag `importBatchId` + możliwość pełnego cofnięcia.

**Goal:** Użytkownik wrzuca plik CSV wyeksportowany ze Strong albo Hevy, widzi podgląd (ile treningów, zakres dat, które ćwiczenia zmapowane automatycznie, które wymagają ręcznego wskazania), zatwierdza i dostaje pełną historię w apce: treningi w Historii, rekordy, wykresy progresji, heatmapa. Ponowny import tego samego pliku niczego nie duplikuje. Import da się cofnąć jednym przyciskiem. Release train na końcu.

**Architektura podejścia:**
- **Parser w 100% po stronie klienta** (plik nie opuszcza urządzenia poza zapisem zmapowanych treningów do własnego Firestore usera). Formaty: Strong CSV (kolumny typu `Date;Workout Name;Exercise Name;Set Order;Weight;Reps;Distance;Seconds;Notes`) i Hevy CSV (`title,start_time,exercise_title,set_index,weight_kg,reps,distance_km,duration_seconds,rpe`). Nagłówki wykrywane automatycznie; dokładny format zweryfikować na realnych plikach fixture (patrz Z109.1), nie z pamięci.
- **Architektura snapshot+resolver gra na naszą korzyść:** importowane treningi dostają `exercises[].name` (snapshot) + `dayName` z nazwy treningu w CSV, więc NIE potrzebują żadnego planu ani cyklu. Historia działa od razu.
- **Mapowanie nazw:** auto-match do biblioteki 241 ćwiczeń (normalizacja + słownik EN→PL najpopularniejszych nazw Strong/Hevy), reszta przez ręczny mapper w UI albo "importuj jako własne ćwiczenie".
- **Idempotencja:** `importBatchId = hash(pliku)`, doc id treningu = `imported-<batchId>-<n>`. Ponowny import = te same id = nadpisanie samego siebie, zero duplikatów. Cofnięcie = delete wg `importBatchId`.

**Tech stack:** React + TS, vitest (fixtures CSV w `src/test/fixtures/`), Firestore rules + `test:rules`, Playwright mock.
