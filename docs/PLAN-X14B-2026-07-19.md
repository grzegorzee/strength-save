**Goal:** Ćwiczenia mają typ śledzenia (ciężar x powtórzenia, czas, czas+dystans+ciężar, asysta), UI serii renderuje właściwe pola, PR/tonaż/1RM liczą się poprawnie per typ (dla asysty: obciążenie efektywne = masa ciała minus asysta), a przy polu ciężaru działa kalkulator talerzy i generator serii rozgrzewkowych z %1RM. Release train na końcu.

**Architektura podejścia:**
- **`SetData` rozszerzone TYLKO polami opcjonalnymi** (`src/types/index.ts:10`): `durationSec?`, `distanceM?`, `assistWeight?`. Stare treningi i plany działają bez zmian (zero migracji).
- **Typ śledzenia per ćwiczenie:** `LibraryExercise.tracking?: 'weight_reps' | 'bodyweight_reps' | 'duration' | 'weight_distance_duration' | 'assisted_bodyweight'` (`src/data/exerciseLibrary.ts:1`). Brak pola = `weight_reps` (default, spójny z obecnym zachowaniem; `isBodyweight` mapuje na `bodyweight_reps`). Własne ćwiczenia: wybór typu w formularzu (`useCustomExercises`).
- **Asysta:** obciążenie efektywne `effectiveLoad = bodyWeight - assistWeight`, masa ciała z najnowszego pomiaru `measurements` (fallback: brak pomiaru = PR tylko po powtórzeniach + zachęta do uzupełnienia wagi). Mniejsza asysta przy tych samych powtórzeniach = NOWY PR.
- **Kalkulatory = czyste funkcje + małe komponenty.** Zero zależności od backendu. Inwentarz talerzy w localStorage (per urządzenie, v1 bez syncu, prostota).

**Tech stack:** React + TS, vitest, Playwright mock, Firestore rules (schemat workouts) + `test:rules` (JDK21).
