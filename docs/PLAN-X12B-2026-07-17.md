> Kontekst: wymaga WDROŻONEGO planu X12A (docs/PLAN-X12A-2026-07-17.md). Decyzje produktowe usera z 2026-07-17: (1) Adaptive Coach wylatuje ("belka na dashboardzie nic nie robi"), (2) hamburger na mobile wylatuje ("zupełnie niepotrzebny"), (3) wersjonowanie: od tego wydania aplikacja ma wersję 1.0.0 i trzyma ją do launchu.

**Goal:** Mniej UI, które nic nie robi: Dashboard bez belki Adaptive Coach, mobile bez hamburgera i drawera (wszystkie miejsca docelowe osiągalne inną drogą), aplikacja w wersji 1.0.0 (package.json + iOS MARKETING_VERSION + Android versionName) z zapisaną zasadą wersjonowania do launchu. Na końcu release train: web + iOS na TestFlight jako 1.0.0.

**Architektura podejścia:** Najpierw inwentarze (co dokładnie wycinamy, co musi pozostać osiągalne), potem wycinka z gwarancjami: rg zero referencji po usunięciu, e2e osiągalności tras na mobile. Wersja 1.0.0 na końcu fazy porządkowej, żeby release poszedł już z nową numeracją.

**Tech stack:** React + TypeScript + Vite, Capacitor iOS/Android, vitest + Playwright (mock), i18n PL/EN.
